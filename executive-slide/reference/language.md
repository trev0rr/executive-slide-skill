# Language — Action Titles, Copy Rules, and the Humanizer Pass

Slide copy is executive prose: short, active, specific, and written the way a partner speaks to a CEO. Fewer words, more impact — brevity drives clarity.

## 1. Action titles

Every body slide's title is a **complete sentence stating the "so what"** — the conclusion the audience should draw, not the topic of the slide.

Rules:
- Full sentence with a subject and an active verb ("build," not "was built"). Ends with a period.
- One key idea per slide; the title asserts it. Quantify when the evidence allows.
- ≤2 lines at 28px. Cut every word that doesn't carry meaning.
- Never a label ("Market Size," "Q3 Financial Overview," "Next Steps"), never a question, never a hedge ("Some considerations regarding…").

Bad → good pairs:

| Bad (label / buried meaning) | Good (so-what sentence) |
|---|---|
| "Q3 Financial Overview" | "Q3 revenue beat forecast by 15%, driven by APAC expansion." |
| "Market Size" | "The $50B residential moving market is fragmented, with no national player." |
| "Growth Trends" | "50% annual growth with declining profitability is unsustainable — we need a strategy that balances both." |
| "Implementation Timeline" | "Three projects deliver the strategy over six quarters." |
| "Financial Projections" | "The profitable-growth strategy doubles shareowner value in five years, adding $5.4B of market capital." |

Common title failures: titles vs. headlines (labels instead of claims), flow problems between slides (each title written in isolation), buried meaning (the claim hidden in the second clause), long sentences, imprecise language, stating the obvious.

## 2. Copy rules (all slide text)

- **Active voice**: "We estimate…" not "It was estimated…"
- **Audience's terms**: use the client's company and industry vocabulary, not consulting-speak.
- **No jargon, no stacked buzzwords**: "We formulated your AI-powered supply chain strategy leveraging industry best practices…" is the canonical failure. One plain verb beats three impressive ones.
- **Positive framing where honest**: state the opportunity, not only the problem — without spinning away real risk.
- **Don't state the obvious**: every line must add a fact, a number, or a consequence.
- **Consistent style**: same tone, tense, person, and punctuation conventions across all slides. Parallel structure within any list (all fragments or all sentences; all verb-first or all noun-first).
- Bullets: ≤ ~8 words, no sub-bullets deeper than one level, no bullet that merely restates its header.

## 3. The humanizer pass

Run this pass over **all slide copy** (titles, headers, bullets, annotations) after drafting and before the self-check. Its job: strip AI-isms and filler so the copy reads like an experienced consultant wrote it.

### Banned words and phrases

delve, leverage / leveraging (use "use"), robust, seamless / seamlessly, holistic, synergy / synergies, cutting-edge, state-of-the-art, empower, unlock, unleash, supercharge, elevate, streamline (as filler), transformative, game-changing, innovative (as a bare adjective), best-in-class, world-class, next-generation, dynamic (as filler), landscape ("the competitive landscape" → name the competitors), journey ("transformation journey"), ecosystem (unless literally a platform ecosystem), "in today's fast-paced / ever-evolving world," "it's important to note," "at the end of the day," "moving forward" (as filler), "a wide range of," "various stakeholders," "drive value / value-add" (say which value), "actionable insights," "key learnings," "deep dive" (as copy — fine as a deck type), "win-win," "low-hanging fruit."

### Banned patterns

- **Bullet soup**: three near-identical bullets that each restate the header with different filler. Each bullet must carry distinct evidence.
- **Empty triads**: "faster, better, stronger" constructions with no numbers behind any of the three.
- **Fragment chains**: em-dash- or arrow-connected fragments in place of a sentence ("Growth → margin pressure → risk").
- **Symmetric fluff**: forcing every column to have exactly N bullets by padding the short ones.
- **Hedge stacking**: "could potentially help enable" → "will cut," "cuts," or delete the claim.
- **Title-case Bullet Headers Everywhere**: sentence case except defined headers.

### Rewrite tests

Apply to every line; rewrite until each passes:
1. **Partner test** — would a senior partner say this sentence aloud to a CEO? If it sounds like a press release or a chatbot, rewrite.
2. **Evidence test** — does the line contain a fact, number, name, or consequence? If not, cut or merge it.
3. **Verb test** — is the main verb concrete ("cuts," "doubles," "wins") rather than abstract ("enables," "supports," "drives")?
4. **Deletion test** — if deleting the line changes nothing, delete it. Whitespace beats filler.
